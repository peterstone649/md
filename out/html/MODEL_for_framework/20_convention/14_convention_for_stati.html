Usage Guidelines

When to Use Each Status:

OPEN: - Task just created - Not yet assigned - Waiting for resources - Planning phase

IN PROGRESS: - Actively being worked on - Has active development - Has recent commits/changes

PAUSED: - Blocked by dependency - Waiting for external input - Temporarily deprioritized - Resource constraints

REVIEW: - Work complete - Awaiting validation - Pending peer feedback - Quality check pending

APPROVED: - Work reviewed and approved - Passed all validation checks - Ready for final implementation - Awaiting completion/finalization

DONE: - All requirements met - Validation passed - No remaining action items - Ready for production/use

ARCHIVED: - Historical record - Reference purposes only - No further changes expected - Completed more than 30 days ago

Status Transition Rules

  1. OPEN ‚Üí IN PROGRESS: When work begins
  2. IN PROGRESS ‚Üí PAUSED: When work halts temporarily
  3. IN PROGRESS ‚Üí REVIEW: When work is complete
  4. PAUSED ‚Üí IN PROGRESS: When work resumes
  5. PAUSED ‚Üí ARCHIVED: When work is abandoned
  6. REVIEW ‚Üí APPROVED: When validation passes and work is approved
  7. REVIEW ‚Üí IN PROGRESS: When feedback requires changes
  8. APPROVED ‚Üí DONE: When approved work is finalized/completed
  9. APPROVED ‚Üí IN PROGRESS: When approved work needs further changes
  10. DONE ‚Üí ARCHIVED: After 30 days of stability
