## Overview

**We establish standardized prefix conventions that enable consistent categorization, programmatic processing, and systematic organization across the entire framework ecosystem.**

---

## Core Prefix Principles

### **Standard Prefix Format**
```
[TYPE_PREFIX]_[DESCRIPTIVE_NAME]
```

### **Prefix Component Definitions**
- **[TYPE_PREFIX]**: Standardized prefix identifier (lowercase with underscore)
- **[DESCRIPTIVE_NAME]**: Clear, concise description using underscores

### **Prefix Rules**
1. **Format**: All prefixes end with underscore (_)
2. **Case**: All prefixes use lowercase letters
3. **Uniqueness**: Each prefix must be unique within its category
4. **Consistency**: Prefixes follow established naming patterns
5. **Documentation**: All prefixes must be documented with clear descriptions

### **Prefix Application Rules**
1. **Single Prefix**: Each document uses exactly one primary prefix
2. **Category Consistency**: Prefixes must match document content category
3. **File Naming Integration**: Prefixes integrate with file naming conventions
4. **Semantic Accuracy**: Prefix must accurately reflect document purpose
5. **Future Compatibility**: Prefixes designed for framework evolution
