## Active Voice Checklist

### **Sentence-Level Assessment**
- [ ] Subject performs action clearly (active voice)
- [ ] No unnecessary passive constructions
- [ ] Actor identified in technical explanations
- [ ] Philosophical operations shown actively
- [ ] Action verbs preferred over state verbs

### **Document-Level Validation**
- [ ] Active voice ratio meets 80% target
- [ ] Passive voice used only when actor unknown
- [ ] Technical processes show clear agency
- [ ] Philosophical arguments demonstrate action
- [ ] Reader can identify who does what

### **Common Passive Constructions to Avoid**
- [ ] "is performed by" → "performs"
- [ ] "was developed by" → "developed"
- [ ] "are managed through" → "manage"
- [ ] "is validated by" → "validates"
- [ ] "were established by" → "established"

---

## Nominalization Elimination Checklist

### **Detection Assessment**
- [ ] Noun forms of verbs identified (-tion, -ment, -ance)
- [ ] Abstract nouns converted to concrete actions
- [ ] Process nouns changed to active verbs
- [ ] Unnecessary abstractions removed

### **Conversion Validation**
- [ ] Nominalizations replaced with active verbs
- [ ] Clarity improved through conversion
- [ ] Philosophical precision maintained
- [ ] Technical accuracy preserved

### **Common Nominalizations to Convert**
- [ ] "implementation" → "implement"
- [ ] "validation" → "validate"
- [ ] "development" → "develop"
- [ ] "organization" → "organize"
- [ ] "application" → "apply"

---

## Concrete Language Enhancement Checklist

### **Abstract Concept Identification**
- [ ] General terms identified (system, process, framework)
- [ ] Vague descriptions located
- [ ] Conceptual generalizations found
- [ ] Imprecise terminology noted

### **Concrete Replacement Validation**
- [ ] Abstract terms replaced with specific examples
- [ ] Concrete illustrations added
- [ ] Reader can visualize concepts
- [ ] Technical concepts made tangible

### **Example Integration**
- [ ] Each abstract concept has concrete illustration
- [ ] Examples are relevant and clear
- [ ] Multiple examples provided for complex concepts
- [ ] Examples support rather than distract from main points

---

## Sentence Length Optimization Checklist

### **Length Assessment**
- [ ] Sentences under 20 words prioritized
- [ ] Long sentences (>25 words) identified
- [ ] Compound sentences evaluated for splitting
- [ ] Complex structures checked for clarity

### **Splitting Validation**
- [ ] Long sentences split at logical breaks
- [ ] Each resulting sentence has clear focus
- [ ] Transitions added between split sentences
- [ ] Meaning preserved through splitting

### **Rhythm and Flow**
- [ ] Sentence variety maintained (short, medium, long)
- [ ] No more than 2 long sentences per paragraph
- [ ] Complex ideas broken into digestible units
- [ ] Reading flow improved through length variation
