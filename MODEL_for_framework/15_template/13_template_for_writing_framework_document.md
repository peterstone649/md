## Document Structure Template

### **1. Header Section** **[HEADER]**

#### **Title** **[TITLE]**
**Format:** `# [DOCUMENT_TITLE_DESCRIPTIVE] [TYPE_FRAMEWORK_TITLE_ABBREV] **[PRIO: XXX]**`
**Guidelines:**
- State main concept first
- Use active construction when possible
- Include key qualifiers for specificity
- **Mandatory bolding** for priority tag: `**[PRIO: XXX]**`
- **Square brackets** for abbreviation: `[TYPE_FOR_FW_PURPOSE]`

**Examples:**
- **Good:** `# Active Voice [RULE_FOR_MFW_ACTIVE_VOICE] **[PRIO: HIGH]**`
- **Avoid:** `# Rule for Active Voice (RULE_ACTIVE_VOICE)`

#### **Metadata** **[METADATA]**
```
**Framework Integration:** [Specific integration statement]
**Version:** V[Major].[Minor].[Patch] **Date:** YYYY-MM-DD
```

#### **Abstract/Summary** **[ABSTRACT]**
**Purpose:** Concise overview using old-to-new information flow
**Structure:**
- What document covers (old information)
- Key findings/contributions (new information)
- Reader benefit statement

**Word Count:** 50-100 words
**Style:** Active voice, concrete examples, clear transitions

### **2. Introduction Section** **[INTRODUCTION]**

#### **Context Establishment** **[CONTEXT]**
**Purpose:** Connect to existing knowledge using old-to-new flow
**Content:**
- What readers already know
- Why this topic matters now
- How this document advances understanding

#### **Purpose Statement** **[PURPOSE]**
**Format:** Clear topic sentence followed by supporting explanation
**Structure:**
- Main purpose (topic sentence)
- Specific objectives
- Expected outcomes

#### **Scope Definition** **[SCOPE]**
**Content:**
- What document covers
- What document does not cover
- Boundaries and limitations

### **3. Main Content Section** **[CONTENT]**

#### **Section Organization** **[ORGANIZATION]**
**Principle:** One main idea per section with clear topic sentences
**Structure:**
- Topic sentence stating main idea
- Supporting evidence/examples
- Clear transitions to next section

#### **Clarity Techniques** **[CLARITY]**

##### **Character-Action-Subject Application**
**Process:**
1. **Identify Action:** What is actually happening?
2. **Name Actor:** Who/what performs the action?
3. **Position Subject:** Put actor first in sentence

**Template:**
```
Unclear: "The validation of the concept was performed by the framework"
Clear: "The framework validates concepts through systematic verification"
```

##### **Nominalization Elimination**
**Process:**
- Identify noun forms of verbs (-tion, -ment, -ance)
- Convert back to active verbs
- Ensure clarity improvement

**Template:**
```
Nominalized: "The implementation of the solution requires consideration"
Active: "We implement the solution, so we must consider requirements"
```

##### **Concrete Examples**
**Structure:**
- Abstract concept statement
- Concrete example illustration
- Connection back to concept

**Template:**
```
Abstract: "The framework provides systematic validation"
Concrete: "The framework validates theorems using three steps: premise verification, logical operation checking, and conclusion validation"
```

#### **Coherence Maintenance** **[COHERENCE]**
**Techniques:**
- **Old-to-New Flow:** Familiar information precedes unfamiliar
- **Transitions:** Explicit connectors (therefore, however, consequently)
- **Logical Progression:** Clear relationship between ideas

### **4. Validation Section** **[VALIDATION]**

#### **Quality Checklist** **[CHECKLIST]**
- [ ] Active voice used in 80% of sentences
- [ ] No unnecessary nominalizations
- [ ] Concrete examples for abstract concepts
- [ ] Clear topic sentences for each section
- [ ] Old-to-new information flow maintained
- [ ] Transitions between all major ideas

#### **Peer Review Guidelines** **[REVIEW]**
**Reviewers Check:**
- Clarity: Can complex ideas be understood easily?
- Coherence: Do ideas flow logically?
- Concision: Are unnecessary words eliminated?
- Precision: Are terms defined and examples concrete?

### **5. Conclusion Section** **[CONCLUSION]**

#### **Summary of Key Points** **[SUMMARY]**
**Structure:** Restate main contributions using different words
**Focus:** What readers should remember
**Format:** Concise, active voice statements

#### **Implications** **[IMPLICATIONS]**
**Content:**
- Practical applications
- Future research directions
- Framework integration points

#### **Final Statement** **[CLOSING]**
**Purpose:** Memorable closing that reinforces main message
**Style:** Clear, direct, forward-looking

### **6. References/Appendices** **[REFERENCES]**

#### **Integration References** **[INTEGRATION]**
- Related framework documents
- Source materials cited
- Cross-references to other fields

#### **Appendices** **[APPENDICES]**
- Detailed examples
- Technical specifications
- Extended validations

## Writing Style Guidelines Integration

### **Sentence-Level Guidelines** **[SENTENCE_STYLE]**

#### **Active Voice Priority**
**Rule:** Use active voice unless passive is required for emphasis

#### **Concision Standards**
**Rule:** Each word must contribute unique meaning

#### **Concrete Language Requirement**
**Rule:** Use specific, tangible terms that create mental images

### **Paragraph-Level Guidelines** **[PARAGRAPH_STYLE]**

#### **Topic Sentence Requirement**
**Rule:** Every paragraph begins with clear statement of main idea

#### **Coherence Standards**
**Rule:** Ideas connect logically with explicit transitions

#### **Unity Principle**
**Rule:** One main idea per paragraph

### **Document-Level Guidelines** **[DOCUMENT_STYLE]**

#### **Old-to-New Flow**
**Rule:** Familiar information precedes unfamiliar throughout document

#### **Logical Progression**
**Rule:** Ideas build systematically from foundation to application
