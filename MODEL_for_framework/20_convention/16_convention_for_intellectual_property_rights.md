# Intellectual Property Rights [CONV_FOR_MFW_IPR] **[PRIO: HIGH]**

**Version: V1.0.1** **Status: APPROVED** **Date: 2026-01-23**
**Scope:** All framework documents, code, and derivative works

## Derivation Source

Based on EUPL v1.2 license framework and international IP standards

## Core Inheritance

- Inherits fundamental IP principles from EUPL v1.2 framework
- Adapts copyright and licensing for framework methodology requirements
- Extends attribution and ownership for collaborative development
- Maintains compatibility with open science and academic publishing standards

## Rationale

Clear intellectual property rights conventions are essential for protecting creators, enabling collaboration, and ensuring legal compliance. This convention establishes transparent ownership, licensing, and attribution practices that support both individual rights and collective innovation within the framework ecosystem.

## General Reference

This convention applies to all framework documents, code, and derivative works. It ensures legal clarity while promoting open collaboration and innovation.

## Abstract

This convention establishes comprehensive intellectual property rights guidelines for the framework. It defines ownership, licensing, attribution, and usage rights for all framework components, ensuring legal protection while enabling collaborative development and open innovation.

## 1. Scope and Applicability

### 1.1 When to Apply
This convention applies to:
- All framework documents and content
- Code, scripts, and software components
- Derivative works and adaptations
- Commercial and non-commercial usage
- Academic and research applications

### 1.2 Target Audience
- Framework developers and contributors
- Commercial implementers and users
- Academic researchers and institutions
- Open source community members

## 2. Core Definitions

| Element | Definition | Example |
|---------|------------|---------|
| **Copyright** | Legal protection for original creative works | Framework documents, code |
| **License** | Permission to use copyrighted material | EUPL v1.2, MIT, Apache 2.0 |
| **Attribution** | Credit given to original creators | © 2026 Raoul Menzel |
| **Derivative Work** | Modified or adapted original work | Framework extensions, implementations |
| **Fair Use** | Limited use without permission | Research, education, criticism |

## 3. Intellectual Property Rights

### 3.1 Copyright Ownership
- **Framework Core**: Copyright held by original creator (Raoul Menzel)
- **Contributions**: Contributors retain copyright to their additions
- **Licensing**: All contributions licensed under compatible open licenses
- **Attribution**: Clear attribution required for all uses and derivatives

### 3.2 License Framework
| License Type | Usage Rights | Requirements |
|--------------|--------------|--------------|
| **EUPL v1.2** | Full usage, modification, distribution | Attribution, license copy, share-alike |
| **Compatible Licenses** | Apache 2.0, MIT, BSD | Attribution, license compatibility |
| **Proprietary Extensions** | Commercial implementations | Framework attribution, compatibility |

### 3.3 Attribution Requirements
- Include copyright notice in all copies and derivatives
- Reference original framework and contributors
- Maintain license text accessibility
- Preserve attribution chains in collaborative works

## 4. Rules and Guidelines

### 4.1 Copyright Protection Rules
- **Rule 1**: All original framework content is protected by copyright
- **Rule 2**: Copyright notices must be preserved in all copies and derivatives
- **Rule 3**: Commercial use requires license compliance and attribution
- **Rule 4**: Academic use requires proper citation and attribution
- **Rule 5**: Modifications must maintain license compatibility

### 4.2 Licensing Guidelines
- **Guideline 1**: Use EUPL v1.2 for framework core components
- **Guideline 2**: Choose compatible licenses for extensions and derivatives
- **Guideline 3**: Document license choices and rationales
- **Guideline 4**: Ensure license compatibility across framework ecosystem
- **Guideline 5**: Provide clear licensing information for users

### 4.3 Attribution Guidelines
- **Guideline 6**: Include full attribution in prominent locations
- **Guideline 7**: Use standardized attribution formats
- **Guideline 8**: Maintain attribution in digital and physical formats
- **Guideline 9**: Credit all contributors appropriately
- **Guideline 10**: Preserve attribution chains in collaborative works
